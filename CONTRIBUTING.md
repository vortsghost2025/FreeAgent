# Contributing to FreeAgent

We welcome contributions that align with the project's goals and constraints.

## How to Contribute

- **Report issues**: Use GitHub issues to report bugs or request features. Include steps to reproduce and relevant logs.
- **Submit changes**: Fork the repository, create a branch, and submit a pull request. Keep changes focused and well-documented.
- **Code review**: All changes require review. Discuss significant changes in issues before implementing.

## Development Setup

1. Clone the repository.
2. Install dependencies (Python, Node.js) as needed for the components you are working on.
3. Run existing tests to ensure your changes do not break functionality.

## Testing

- Run unit tests before submitting code.
- Add tests for new functionality.
- See `TESTING.md` for detailed testing instructions.

## Constraints

- Do not introduce runtime enforcement of new governance rules without explicit design review.
- Do not move or delete existing source paths without approval.
- Do not enable feature flags or Phase 2 features unless explicitly authorized.

## Code Style

- Follow existing code style in each language (Python, JavaScript, Markdown).
- Keep functions small and focused.
- Document non‑obvious decisions.

## Communication

- Use GitHub issues and pull request discussions for project coordination.
- For sensitive security issues, use the SECURITY.md process.
