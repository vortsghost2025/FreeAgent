# ARCHITECTURE GUIDE

This guide provides an overview of the system architecture, including major subsystems, data flows, and design principles.

## Subsystems
- Narrative Generation
- Multi-Federation Politics
- Persistent Universe
- Anomaly Engine
- Federation Dashboard (UI)
- Mobile Extension (offline-first, mesh-aware)

## Data Flow
- Events trigger narrative and political updates
- Universe state is persisted and loaded via documentation-driven mechanisms

## Design Principles
- Modularity
- Extensibility
- Robustness
- Documentation-driven persistence

## Extension Points
- Mobile app integration
- Real-world multi-device testing
- Cross-platform support
