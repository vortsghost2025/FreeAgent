# Project Architecture Blueprint Generation Plan

**Goal**: Produce a comprehensive `Project_Architecture_Blueprint.md` that analyzes the current codebase, detects technologies and architectural patterns, and documents the architecture with diagrams, decision records, code examples, and extensibility guidance.

## 3. Research & Discovery Phase
1. **Technology Detection**
   - Scan for language‑specific project files (`*.csproj`, `pom.xml`, `package.json`, `requirements.txt`, `pubspec.yaml`, etc.).
   - Parse dependencies to infer frameworks (ASP.NET Core, Spring, React, Angular, Flask, etc.).
   - Identify build tools (MSBuild, Maven/Gradle, npm/Yarn, Dockerfiles, CI pipelines).
2. **Architecture Pattern Detection**
   - Analyse folder hierarchy and naming conventions (e.g., `Domain`, `Application`, `Infrastructure`, `Controllers`, `Services`).
   - Examine dependency graphs (imports, `using`, `import`, `require`).
   - Look for patterns: Repository, Service, Mediator, CQRS, Event‑Driven, etc.
3. **Diagram Type Decision**
   - If `DIAGRAM_TYPE` is C4, plan to generate Mermaid C4 diagrams.
   - If UML, plan to generate PlantUML class/component diagrams.
   - Determine tooling: Mermaid (inline) or generate PNG/SVG via external CLI (e.g., `mmdc`).
4. **Decision Record Extraction**
   - Search for `docs/adr`, `architecture/decisions`, or markdown files with `# Decision` headings.
   - If none, infer decisions from commit messages and code comments.

## 4. Data Collection Steps (Read‑Only)
- **Glob** patterns to locate relevant files:
  - `**/*.csproj`, `**/*.sln`
  - `**/pom.xml`, `**/build.gradle*`
  - `**/package.json`, `**/yarn.lock`, `**/pnpm-lock.yaml`
  - `**/*.py`, `**/requirements.txt`
  - `**/Dockerfile*`, `**/*.dockerignore`
  - `**/README.md`, `**/docs/**/*.md`
- **Read** each discovered file to extract:
  - Language/framework identifiers.
  - Architectural layer markers (e.g., folder names, namespaces).
  - DI container registrations.
  - API route definitions.
  - Configuration sources (`appsettings.json`, `application.yml`, `.env`).
- **Grep** for key patterns:
  - `@Controller`, `@RestController`, `@Service`, `@Repository` (Java/Spring).
  - `services.Add*`, `builder.Services` (ASP.NET).
  - `store.configure`, `createStore` (React/Redux).
  - `@Injectable`, `providers` (Angular).
  - `@dataclass`, `pydantic` (Python).
- **Collect** cross‑cutting concerns (auth, logging, error handling) by grepping for common libraries (`Serilog`, `log4j`, `winston`, `loguru`, `jwt`, `oauth`).
