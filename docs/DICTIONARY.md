# Dictionary / Glossary

This file defines key terms and acronyms used in FreeAgent and related documentation.

## Terms

- **FreeAgent** — The active runtime/orchestration workspace and implementation trunk. Not a constitutional lane.

- **4‑lane lattice** — The constitutional governance framework that holds supreme authority over rules, constraints, and enforcement. FreeAgent defers to this authority.

- **Lane** — A constitutional verification or governance lane within the lattice (e.g., conservative, adversarial, coordination, enforcement). FreeAgent is not a lane.

- **Nexus Graph** — A visualization of document relationships, verification states, and contradictions. It is a descriptive aid, not a source of constitutional authority.

- **Compact / compact snapshot** — A deterministic, compressed serialization of system state used for continuity and restore operations.

- **Phenotype continuity** — A measure of similarity between runtime state and a known-good reference state (often used for continuity checks).

- **Authoritative field** — A data element used for enforcement decisions (e.g., cryptographic hash, continuity probe result).

- **Advisory field** — Informational metadata that does not affect enforcement (e.g., human notes, non‑verified timestamps).

- **NFM (Need‑for‑More)** — An observed gap, contradiction, or missing constraint that motivates discovery or policy refinement.

- **Delegation surface** — The set of external services, agents, or APIs that the system depends on or exposes.

- **Constitutional authority** — The ultimate source of rules and enforcement rights. In this ecosystem, held by the 4‑lane lattice.

- **Operational guide** — Documentation that advises runtime behavior but does not define constitutional law (e.g., AGENTS.md in FreeAgent).

- **Policy fragment** — A proposed or ratified rule that may constrain system behavior once ratified by the lattice.

- **Enforcement reachability** — Whether a constraint can be actively enforced at runtime by the system.

- **Self‑organizing library** — A knowledge/verification surface in the lattice that maintains authoritative definitions and relationships.

- **WE4Free** — A public‑impact mental‑health resource hub and associated tooling (web, mesh, papers).

- **Phase 1 extraction** (completed) — Extraction of shared‑infra, federation‑creative, and connection‑bridge into independent repositories.

- **Feature flag** — A runtime configuration toggle used to enable/disable functionality (e.g., continuity gate) without code changes.
